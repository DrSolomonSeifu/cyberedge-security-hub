7A

Topic 7A — Asset Management

You cannot protect what you do not know you have. Asset management is the foundation that makes every other security control more effective — it answers what exists, where it is, and how much it matters.

7AIntroduction

Asset Management — Introduction

Asset management involves identifying, classifying, and inventorying all of an organization's IT assets — hardware, software, data, and personnel. It is the foundation that makes every other security function more effective.

🖥️
HARDWARE

Servers, workstations, laptops, mobile devices, network equipment. If it connects to the network, it is an asset that must be tracked and secured.

💿
SOFTWARE

Applications, operating systems, licenses, installed tools. Software inventory enables patch management and license compliance.

🗄️
DATA

Databases, files, intellectual property, sensitive records. Data classification drives protection decisions — not all data carries equal risk.

👥
PERSONNEL

Who has access to what, and at what privilege level. Access tracking is part of the asset management discipline.

🔑 What Effective Asset Management Enables
  • A clear understanding of the organization's entire IT infrastructure — what exists, where, and who is responsible
  • Continuous monitoring for unauthorized activity — an unrecognized device on the network is immediately suspicious
  • Identification of potential vulnerabilities — unpatched systems and outdated software are visible only when assets are tracked
  • Rapid incident response — teams identify and isolate affected assets before the incident spreads
An untracked server running in a forgotten corner of the data center — not patched, not monitored, still connected — is not hypothetical. It is one of the most common real-world security gaps. Asset management closes it.
7AAsset Tracking

Asset Tracking, Assignment & Enumeration

Tracking goes beyond knowing what assets exist — it tracks performance, security status, usage, and location continuously. Ownership assigns accountability to every asset before an incident forces the question.

Asset Database
A centralized record of every asset. Typical fields: type, model, serial number, asset ID, location, assigned user, monetary value, and service history. Stores as much or as little detail as the organization needs.
RFID Tag
A chip programmed with asset data. When in range of a scanner, the chip activates and signals the scanner — which updates the asset's location in management software. Makes theft more difficult and enables real-time location tracking.
Naming Convention
A standard naming strategy that lets any administrator immediately identify a resource's type, function, and location from its name alone. Applies to hardware, accounts, virtual machines, DNS records, and network resources.
CMDB
Configuration Management Database — a centralized repository of IT infrastructure information. Tracks assets, configurations, and relationships between systems. Critical for understanding how a change to one asset affects everything connected to it. Tools: ServiceNow, BMC Remedy.
MANUAL INVENTORY

Physically inspecting assets and recording details. Practical for smaller organizations. Time-intensive and prone to human error at scale.

NETWORK SCANNING

Tools like Nmap, Nessus, or OpenVAS automatically discover networked devices. Identifies open ports, services, and operating systems. Fast and scalable.

ASSET MGMT SOFTWARE

Lansweeper, ManageEngine, SolarWinds. Automatically discover, track, and catalog hardware, software, and licenses. Centralized dashboard.

MDM SOLUTIONS

Microsoft Intune, VMware Workspace ONE. Enumerates, manages, and secures mobile devices. Essential for BYOD environments.

CLOUD DISCOVERY

AWS Config, Azure Resource Graph, CloudAware. Discovers and catalogs assets deployed across cloud environments. Cloud assets need tracking too.

SECURE PROCUREMENT

Security must be built into the purchase decision — not added after deployment. Total Cost of Ownership (TCO) includes maintenance, update costs, and potential incident costs.

Asset classification drives prioritization — a vulnerability on a critical production database outranks the same vulnerability on a test system. Without classification, everything looks equally important and nothing is properly protected.
7AAsset Protection

Asset Protection Concepts

Asset protection is about ensuring every configurable element in the inventory has not drifted from its approved, secure state — and that all changes happen in a controlled, documented way.

Configuration Item (CI)
Under ITIL, a CI is an asset that requires specific management procedures to deliver a service. Every CI must be labeled (using the standard naming convention), defined by its attributes, and tracked in the CMDB.
Baseline Configuration
A list of settings an asset must adhere to at all times. Security baseline = the minimum set of security configuration settings a device must maintain to be considered adequately protected. Any deviation is a finding requiring remediation.
Configuration Mgmt System (CMS)
The tools and databases used to collect, store, manage, update, and report on CI information. Small networks: spreadsheets and diagrams. Enterprise environments: dedicated applications.
Change Control
Every proposed change goes through a formal review and approval process before implementation. Changes are tested, documented, and scheduled. Unauthorized changes are a major source of misconfigurations.
Network Diagrams
The best way to capture complex relationships between network elements. Document: logical topology (IP addressing, routing), physical topology (cable runs, rack layouts), and business workflows. Without accurate diagrams, both incident response and vulnerability assessment slow down significantly.
ITIL framework: Service Assets → Configuration Items (CIs) → Baseline Configuration → CMS → CMDB. Know this hierarchy. Configuration drift — when systems change over time without proper tracking — is caught only through regular configuration audits comparing current state against the approved baseline.
7AData Backups

Data Backups

If every system the organization owns was encrypted by ransomware tomorrow — what happens next? The answer depends entirely on whether reliable, tested, recoverable backups exist.

✓ Simple Backup Methods

  • Copying files to external drive or basic cloud storage
  • Fast to set up and easy to understand
  • Works for small, low-complexity environments
  • Breaks down at enterprise scale — scalability, performance, recovery time, and granularity all fail

⚡ Enterprise Backup Strategy

  • Support for virtual, physical, and cloud environments
  • Data deduplication and compression
  • Instant recovery and replication for quick failover
  • Ransomware protection and encryption
  • Granular restore — individual files, folders, or applications
  • Reporting, monitoring, and alerting tools
On-Site Backup
Data stored locally — same site as the protected systems. Fast access and rapid recovery for common failures. Vulnerable to the same physical threats as primary systems. Ransomware can reach it if connected to the network.
Off-Site Backup
Data transferred to a remote location — physically separate from the primary site. Protects against natural disasters, theft, and catastrophic local failures. The geographic separation breaks the ransomware chain.
Air-Gapped Backup
Physically disconnected from the network entirely. An air-gapped backup cannot be reached by ransomware under any circumstances. The gold standard for ransomware-resilient backup strategy.
Data Deduplication
Identifies and eliminates redundant data blocks — stores one copy and creates references to it for all other instances. Can operate at file, block, or byte level. Significantly reduces storage requirements and speeds up data transfer.
⚠ Recovery Validation — Never Skip This
  • Full recovery test — restore an entire system to a separate environment and verify that OS, applications, and data all function correctly
  • Partial recovery test — restore selected files, folders, or databases to validate integrity of specific data subsets
  • Backups can complete with 100% reported success and still fail completely when used for actual recovery — testing is the only way to know
  • Recovery times are almost always longer than organizations assume — testing reveals the real number so planning can account for it
Ransomware increasingly targets backup infrastructure specifically. If backups are encrypted by ransomware, recovery becomes impossible. Air-gapped backups that cannot be reached by ransomware are the essential defense.
7AAdvanced Data Protection

Advanced Data Protection — Snapshots, Replication & Journaling

Think of a snapshot as a photograph of a system at a specific moment in time. If something breaks after that photo was taken, you have a record of exactly how it looked before — and you can return to it.

VM SNAPSHOTS

Captures the full state of a virtual machine — memory, storage, and configuration. Used in VMware vSphere and Hyper-V (called checkpoints). Roll back the VM to any prior state after a failed update or change.

FILESYSTEM SNAPSHOTS

Captures the state of a file system at a given moment. Enables recovery of accidentally deleted files or previous versions. Examples: ZFS, Btrfs. Recovery is surgical — restore exactly what was lost.

SAN SNAPSHOTS

Taken at the block-level storage layer. Captures the state of an entire storage volume. Examples: NetApp, Dell EMC. Enables rapid recovery of large datasets and entire applications.

REPLICATION

Creates and maintains exact copies of data on different storage systems or locations. Any change to the primary is automatically copied to the secondary. Protects against hardware failure, human error, and malicious attacks.

JOURNALING

Records every change to data in a separate, dedicated log (a journal). Used in NTFS and JFS. Enables recovery after a system crash — identifies and undoes incomplete transactions. Greater granularity than snapshots.

BACKUP ENCRYPTION

A backup containing sensitive data is a liability if unencrypted. Encryption ensures stolen or intercepted backup data is unreadable. Required by HIPAA, PCI-DSS, and GDPR for regulated data. Many organizations overlook backup sets in security reviews.

Replication TypeDescriptionTrade-off
SynchronousData written to both locations simultaneously — both copies always identicalHigher cost and latency — guarantees consistency
AsynchronousSecondary updated shortly after primary — slight lag between copiesMore cost-effective — small window of potential data loss
SAN ReplicationDuplicates data between two SANs in real time or near real timeEnterprise-scale redundancy for storage-critical environments
VM ReplicationMaintains an up-to-date copy of a VM on a separate host or locationIf the primary VM fails, the secondary takes over workload immediately
7ASecure Data Destruction

Secure Data Destruction

Deleting a file does not destroy the data. The sectors containing the data are marked as available — the data remains until something else overwrites it. Secure data destruction is a formal discipline, not an afterthought.

📋 When Data Destruction Is Required
  • End of data retention period — internal policies and regulations define how long data must be kept. When that period ends, the data must be destroyed.
  • Regulatory compliance — GDPR gives individuals the right to request deletion. HIPAA requires destruction of patient data when no longer needed.
  • Decommissioning storage devices — before any device is disposed of, repurposed, or donated, data must be completely and verifiably destroyed.

🖴 Hard Disk Drive (HDD) Methods

  • Zero filling — sets every bit to zero. Single pass can leave readable patterns with specialist tools.
  • Three-pass overwrite — pass 1: all zeros; pass 2: all ones; pass 3: pseudorandom pattern. More secure.
  • Some federal agencies require more than three passes.
  • Overwriting can take considerable time depending on the number of passes.

💾 Solid-State Drive (SSD) Methods

  • Traditional overwriting is less effective — wear leveling and bad block management leave some cells inaccessible to standard tools.
  • ATA Secure Erase — instructs the drive's own firmware to internally sanitize all stored data, including inaccessible marked-as-bad memory cells.
  • Designed specifically for SSD architecture.
Sanitization
Removing sensitive information from storage media so it becomes irretrievable — even with specialist recovery tools. Used when a device is being repurposed or donated. Methods: data wiping, degaussing, or encryption-based sanitization (encrypt and destroy the key).
Destruction
Physical or electronic elimination of the data — rendering it completely irrecoverable. Physical methods: shredding, crushing, incineration. The most absolute form of data elimination — once shredded, no recovery is possible under any circumstances. Used when a device will not be reused.
Certification
Documentation and formal verification that sanitization or destruction was completed correctly. A certificate of destruction from a reputable third-party provider attests to compliance. Third-party certification carries significantly more weight with regulators and auditors than self-certification.
Choosing the right method — Repurposing internally: sanitization. Donating externally: sanitization + certification. Decommissioning permanently: destruction + certification. High-security environments: physical destruction only.

7B

Topic 7B — Redundancy Strategies

Redundancy strategies are not about preventing failure — they are about designing systems that survive it. When hardware fails, power goes out, or ransomware strikes, redundancy determines whether operations continue or halt.

7BCOOP & Capacity Planning

Continuity of Operations & Capacity Planning

COOP ensures an organization can maintain or quickly resume critical functions during a disruption, disaster, or crisis. It focuses on the immediate response — what must keep running, and how.

🔑 Key Elements of a COOP Plan
  • Identify critical functions — what absolutely must keep running no matter what?
  • Establish priorities — not everything can be restored at once. Most critical functions come first.
  • IT redundancy — off-site backups, failover systems, and disaster recovery solutions
  • Alternative work arrangements — remote work plans and co-location facilities
  • Communication protocols — every employee knows their role and decision-making authority in advance

📋 COOP vs. Business Continuity

  • COOP — immediate response and restoration of critical functions. Shorter time frame — the first hours and days.
  • Business Continuity (BC) — broader, longer-term view. Addresses overall organizational resilience including supply chain, employee safety, legal compliance, and reputation management.
  • COOP is a component of the broader BC concept.

📊 Capacity Planning Dimensions

  • People — staffing levels, skill sets, cross-training, succession planning
  • Technology — hardware, software, and network resources for current and future workloads
  • Infrastructure — physical facilities, data centers, power, cooling, connectivity
  • Methods: trend analysis, simulation modeling, and benchmarking
⚠ Capacity Planning Risks
  • Cross-training — employees must develop skills outside their primary roles. If one key person is unavailable, a critical function should not stop entirely.
  • Layoff risks — disgruntled departing employees may engage in unauthorized access. Revoke system access on the day of departure. Ensure proper knowledge transfer before staff leave.
  • Rapid hiring risks — insufficient vetting and insufficient security awareness in new hires. Access provisioned too broadly under time pressure.
  • Under-planning — overloaded systems become vulnerable to crashes and DoS attacks. Critical patches and security measures get skipped.
7BHigh Availability

High Availability, Scalability & Site Resiliency

High availability ensures systems remain operational and accessible with minimal downtime. It requires designing every layer — hardware components, servers, networking, and data centers — for fault tolerance and redundancy.

NinesAvailabilityAnnual DowntimeTypical Use
Two99%87 hours 36 minNon-critical systems
Three99.9%8 hours 45 minStandard business systems
Four99.99%52 minutes 34 secBusiness-critical systems
Five99.999%5 minutes 15 secMission-critical / telecom
Six99.9999%32 secondsExtreme criticality

📈 Scalability

  • Scale-up (vertical) — add more CPU or memory to an existing resource
  • Scale-out (horizontal) — add more instances working in parallel
  • Elasticity — the system handles demand changes in real time automatically. No service degradation when demand spikes suddenly.

🏢 Site Resiliency Types

  • Hot site — failover almost immediately. Fully operational, data current. Most expensive.
  • Warm site — equipment in place but latest data must be loaded. Balance of cost and speed.
  • Cold site — empty facility with lease in place. Equipment procured when needed. Slowest, lowest cost.
  • Cloud DR — scalable, cost-effective, geographically diverse. Faster deployment than building new sites.
Maximum Tolerable Downtime (MTD) defines how long a specific business function can be down before unacceptable harm results. Fault tolerance = the system continues at the same or similar level of service even when components fail. Geographic dispersion = recovery sites far enough apart that a single regional disaster cannot affect all of them.
7BClustering

Clustering & Virtual IP

Clustering goes beyond load balancing — multiple redundant processing nodes share data with one another and accept connections. To clients, the entire cluster appears as a single server.

Virtual IP (VIP)
A single public IP address shared between two appliances in a cluster — also called a shared or floating address. A redundancy protocol (e.g., CARP — Common Address Redundancy Protocol) runs on a private connection between nodes. A heartbeat mechanism monitors the active node — if it fails, the passive node takes ownership of the VIP immediately.
Active/Passive (A/P)
One node is active (handling all connections). One node is passive (standby). Advantage: performance is not adversely affected during failover — the passive node has full capacity available. Disadvantage: higher costs because passive capacity is unused during normal operations.
Active/Active (A/A)
Both nodes process connections concurrently. Maximum utilization of hardware — no idle capacity. During failover, the surviving node absorbs the failed node's workload — performance degrades temporarily. Better ROI on hardware investment.
N+1 Configuration
A single passive node is shared among multiple active nodes. Example: five active nodes share one passive node. Significantly reduces the number of passive nodes (and cost) required. If two active nodes fail simultaneously, only one can be covered.
Application Clustering
Servers in the cluster communicate session information to one another. If a user is logged in on one instance and that instance fails, the next session starts on another instance — which already has the cookies and login state. The user experiences no interruption.
Know the difference: load balancers distribute traffic between independent servers — they primarily manage web traffic. Clusters provide redundancy and HA for systems like databases and file servers — nodes share data with each other. Both use failover but at different architectural levels.
7BPower Redundancy

Power Redundancy

Every computer system, server, and network appliance requires a stable power supply. Power management deploys layered systems to protect against voltage spikes, surges, brownouts, and complete power failures.

1

Dual Power Supplies (PSUs)

Enterprise-class servers and appliances feature two or more PSUs. If one fails, the other takes over immediately — no downtime, no data loss. Hot plug PSUs can be replaced without powering down the system.

2

Managed PDUs (Power Distribution Units)

Distributes power across all equipment in a rack. Cleans the power signal and protects against spikes and surges. Managed PDUs support remote monitoring — reporting load, switching sockets on/off, and powering on sockets in a specific sequence after an outage.

3

UPS (Uninterruptible Power Supply)

Provides temporary power during a complete power failure. Runtime: minutes for desktop-rated units to hours for enterprise systems. Keeps systems running while generators spin up. Also enables graceful shutdown if no generator is available — preventing data corruption. A UPS is always required — a generator cannot respond fast enough alone.

4

Backup Generators

Provides power to the entire building — often for several days. Fuel options: diesel (shelf life 18 months–2 years), propane (safe storage required), natural gas (may be disrupted by the same disaster as the power outage). Renewable alternatives: solar, wind, hydrogen fuel cells, geothermal.

Transfer switches connect generator power to building circuits — either manually or automatically. Generator power is typically introduced via transfer switches. The UPS must be sized to carry the full load through the transfer process — which takes 10 to 30 seconds for most generators.
7BDiversity & Defense in Depth

Diversity, Vendor Diversity & Defense in Depth

Platform Diversity
Using multiple technologies, operating systems, and hardware or software components across the infrastructure. A single compromised platform does not automatically compromise everything. Attackers must be familiar with multiple platforms and exploit techniques — the complexity itself acts as a deterrent.
Defense in Depth
Multiple layers of security controls at different levels. Each layer compensates for the potential failure of the layer before it. Layers: perimeter security → network segmentation → endpoint security → identity and access management → data encryption → policy and training → physical security. If an attacker defeats one layer, the next layer stops them.
Vendor Diversity
Using multiple vendors for critical systems and services. A single vendor vulnerability cannot take down everything. Also reduces vendor lock-in, promotes innovation through competition, and improves pricing leverage. Regulatory environments may require vendor diversity for compliance.
Multi-Cloud Strategy
Uses services from multiple CSPs simultaneously. A vulnerability or outage at one cloud provider does not compromise the entire infrastructure. Example: primary app infrastructure on Provider A, backup and DR on Provider B, sensitive data on Provider C (advanced security/compliance), content delivery on Provider D (global edge locations).
Vendor diversity prevents operational single points of failure. Defense in depth prevents security single points of failure. Together, they create an environment where no single failure — technical or security-related — cascades into a complete loss.
7BDeception Technologies

Deception & Disruption Technologies

Attackers invest significant time in reconnaissance before launching an attack. Deception technologies waste that time on fake targets — alerting defenders the moment an attacker interacts with them while protecting real systems through misdirection.

HONEYPOT

A decoy system designed to mimic a real server or service. Any interaction is suspicious — legitimate users have no reason to access it. Monitors attacker behavior and gathers intelligence on tactics, techniques, and tools.

HONEYNET

A network of interconnected honeypots simulating an entire organization's network. More extensive and realistic than a single honeypot. Reveals multi-stage attack techniques including lateral movement and privilege escalation.

HONEYFILE

Fake files that appear to contain sensitive information — payroll.xlsx, passwords.txt, customer_data.csv. Placed where an attacker browsing a file system would naturally look. Any access to a honeyfile triggers an immediate alert.

HONEYTOKEN

False credentials, API keys, or login tokens embedded in systems or files. Look legitimate and indistinguishable from real credentials. If an attacker steals and uses a honeytoken, it triggers an immediate alert revealing that credentials have been compromised.

FAKE TELEMETRY

When a host detects port scanning, it returns false data — reporting multiple ports as open when they are not. The attacker spends time investigating ports and services that do not exist. Includes bogus DNS entries and decoy web server directories.

DNS SINKHOLE

Suspect or malicious traffic is rerouted to a different network — such as a honeynet — instead of reaching its intended destination. Defenders analyze the traffic in detail while the attacker believes they are communicating normally.

Legitimate users have no reason to access honeypots, honeyfiles, or honeytokens — any interaction is immediately suspicious. The goal of all deception technologies is the same: raise the cost of reconnaissance, waste attacker resources, and alert defenders without the attacker knowing they have been detected.
7BTesting Resiliency

Testing Resiliency & Documentation

A resilience plan that has never been tested is a plan that cannot be trusted. Testing reveals the real-world performance of redundancy systems — not their theoretical performance. Untested systems fail exactly when they are needed most.

1

Tabletop Exercises

Discussion-based session where key personnel walk through a simulated disruption scenario. No systems are taken down. Reveals: knowledge gaps, communication gaps, and coordination failures between teams before a real incident exposes them.

2

Failover Tests

A primary system is intentionally taken offline under controlled conditions. The redundant system must take over automatically and correctly. Validates whether failover actually works as designed — and reveals how long it actually takes vs. how long it was assumed to take.

3

Simulations

Full-scale rehearsals of disaster recovery or business continuity scenarios. Teams respond as if a real event is happening. Reveals: the organization's actual preparedness, real recovery times, and undocumented dependencies between systems.

4

Parallel Processing Tests

Primary and backup systems run simultaneously under the same workload. Normal operations are not disrupted. Validates that the backup system can handle the same load as the primary and produces identical results. Reveals capacity gaps and configuration differences.

📄 Documentation Requirements
  • Test plans — objectives, scope, methods, roles, and success/failure criteria defined before the test begins
  • Test scripts and scenarios — step-by-step instructions that make tests repeatable and consistent
  • Test results — what worked, what did not, and what needs to change in the plan
  • Third-party assessments — ISO 22301, PCI DSS, SOC 2 provide independent validation and compliance evidence
Organizations that skip testing face: undetected vulnerabilities in response plans, extended downtime when real incidents occur, data loss, reputational damage, increased recovery costs, and regulatory penalties. Recovery times are almost always far longer than assumed until testing proves otherwise.

7C

Topic 7C — Physical Security

Every digital security control — every firewall, every encryption algorithm, every Zero Trust policy — can be bypassed by one thing: an attacker walking into a room they should not be in. Physical security is the first line of defense.

7CPhysical Security Controls

Physical Security Controls & Security Zones

Physical security protects personnel, hardware, software, networks, and data from physical actions and events that could cause severe damage or loss. It applies the same three access control fundamentals as technical system security.

Authentication
Creates access lists and identifies mechanisms to allow approved persons through the barriers. Who is allowed in? Access badges, biometric scanners, PIN pads, security guard verification.
Authorization
Creates barriers around a resource — walls, doors, fences, cages. Controls access through defined entry and exit points only. Not everyone authenticated to the building is authorized to enter the server room.
Accounting
Records every use of entry and exit points — who entered, when, and for how long. Detects security breaches. Creates an audit trail for post-incident investigation. Access accountability deters unauthorized behavior.

Security zones are the physical equivalent of network segmentation. Progression through zones must become increasingly restrictive. An attacker who breaches the outer zone does not automatically reach the inner zones — each boundary is another opportunity to detect and stop unauthorized access.
7CSite Layout

Site Layout, Fencing, Lighting & Bollards

Physical security does not begin at the server room door — it begins at the outer boundary of the property. Environmental design uses the built environment itself as a security tool.

BARRICADES

Prevents or channels access. No barricade is completely effective — the purpose is to channel people through defined, controlled entry and exit points. Every entry point must have an authentication mechanism. Surveillance covers the perimeter.

FENCING

Three properties: Transparent (guards can see through it), Robust (difficult to cut), Secure against climbing (tall, possibly with razor wire). Gives facilities an industrial appearance — organizations that welcome customers may use more discreet perimeter methods.

SECURITY LIGHTING

Makes intrusion more difficult and surveillance easier. Three functions: perception of safety, deterrence, and enabling camera performance. Design must eliminate shadows, avoid glare, and provide sufficient light for facial recognition where required.

BOLLARDS

Short vertical posts — steel or concrete. Installed at intervals around a perimeter or entrance. Purposes: protect pedestrians, prevent unauthorized vehicle access, protect critical infrastructure. Fixed or retractable. Can appear as sculptures — non-obvious to attackers.

PHYSICAL DMZ

Public access areas positioned so guests never pass near secure zones. Security mechanisms in public areas are highly visible — to deter. Entry points to secure zones are discreet — so attackers cannot locate or inspect them. Industrial camouflage.

SCREEN PROTECTION

Position display screens away from pathways and windows in secure zones. Use one-way glass — visible from inside out only. No one outside can observe credentials being entered or sensitive information displayed.

Vehicular attacks: Physical sites at risk of terrorist attack use bollards and security posts specifically to prevent vehicles from being used as weapons against a building. Government buildings, embassies, airports, and stadiums all employ vehicle-rated bollard systems for this purpose.
7CGateways & Locks

Gateways, Locks & Access Badges

A gateway is only as strong as the lock securing it. Secure gateways must be self-closing and self-locking — not dependent on the user to close and lock behind them. Human behavior is unpredictable; a door that locks itself removes that variable entirely.

Physical Locks
Conventional mechanical locks. Standard locks are vulnerable to picking and bumping. Deadbolt locks extend a bolt directly into the door frame — significantly more resistant to forced entry. Keys can be copied, lost, or stolen — no audit trail of use.
Electronic Locks
Operate using a PIN entered on a keypad. Also called cipher, combination, or keyless locks. Variations: magnetic swipe card, proximity reader (detects wireless key fob), smart lock. PINs and codes can be changed without replacing the lock.
Biometric Locks
Integrated with a biometric scanner — fingerprint, retina, facial recognition. The credential cannot be lost, stolen, or shared. Used in data centers, research facilities, and government installations. The most secure lock type.
Access Control Vestibule (Mantrap)
Two interlocked doors that permit only one individual to pass through at a time. The first door opens after authentication — closes before the second opens. Prevents tailgating and piggybacking. Used in data centers, government buildings, and financial institutions.
Cable Locks
Attaches to a secure point on the device chassis (metal loop or Kensington security slot). Prevents physical theft. The position of the secure point also prevents the chassis from being opened without removing the cable first.
Access Badges & PACS
Plastic cards with magnetic stripe, RFID chip, or NFC chip. Swiped, tapped, or brought into proximity with a reader. The Physical Access Control System (PACS) verifies the badge and logs every access event — time, location, and identity. Also serves as visual identification showing name, title, and photograph.
Tailgating = following an authorized person through a secured door. Piggybacking = slipping through before the door closes. The mantrap prevents both by enforcing one-person-at-a-time entry with interlocked doors. Kensington slot is the standardized slot found on laptops, servers, and monitors for cable lock attachment.
7CGuards & Cameras

Security Guards & Video Surveillance

Surveillance is a second layer of security — reinforcing what perimeter controls already provide. Two primary forms of surveillance exist: human security guards and video surveillance systems (CCTV).

👮 Security Guards

  • Most responsive and intelligent form of surveillance — apply knowledge and intuition
  • Monitor checkpoints, verify identification, allow or deny access, log physical entry events
  • Visible presence is one of the most effective deterrents available
  • Most expensive form of surveillance — personnel costs are ongoing
  • Screening and training are imperative — a poorly vetted guard is an insider threat
  • Some zones cannot have guards due to security clearance requirements

📷 CCTV Systems

  • More affordable than stationing guards at every gateway
  • Movement and access are recorded — footage supports investigations and audits
  • Cameras connected to a multiplexer via coaxial cabling or IP networking
  • Limitation: longer response times — footage must be monitored to trigger a response
  • Effectiveness depends on sufficient staff monitoring feeds. An unwatched camera is a recorder — not a security control.
MOTION RECOGNITION

Gait identification technology — the system learns authorized individuals' movement patterns. Generates an alert when anyone moves in a pattern not matching a known, authorized person. Detects unauthorized individuals before they reach a controlled entry point.

OBJECT DETECTION

AI-powered cameras detect changes to the physical environment. Examples: a missing server, an unknown device connected to a wall port, unattended bags in restricted areas. Triggers real-time alerts when the environment changes unexpectedly.

DRONES / UAV

Cameras mounted on drones extend surveillance coverage to wide outdoor areas, rooftops, and perimeters impractical for fixed cameras. Dynamic repositioning — the drone moves where needed. Used for large campuses, industrial facilities, and critical infrastructure.

7CAlarms & Sensors

Alarm Systems & Sensor Types

Alarms play a dual role — they are both detective controls (alerting personnel when a breach occurs) and deterrent controls (discouraging unauthorized access before it is attempted). Alarms work best when integrated with access control systems, surveillance cameras, and motion sensors.

CIRCUIT ALARM

Sounds when an electrical circuit is opened or closed. Closed-circuit alarm is more secure — cutting the circuit triggers the alarm rather than silencing it. Best for: doors, windows, fences, and perimeter entry points.

MOTION DETECTION

Triggered by movement within a defined area. Sensors: microwave radio reflection (like radar) or passive infrared (PIR) detecting moving heat sources. Best for: spaces not normally occupied — storage areas, server rooms after hours.

NOISE DETECTION

Triggered by sounds picked up by a microphone. AI-backed analysis identifies specific sound types — significantly reduces false positives. Best for: perimeter walls, windows, and server rooms where equipment tampering produces sound.

PROXIMITY ALARM

Uses RFID tags and readers to track tagged objects. If a tagged asset moves toward an unauthorized exit — the alarm triggers. Best for: detecting unauthorized removal of hardware from server rooms and equipment areas.

DURESS ALARM

Manually triggered by staff who are under threat or coercion. Methods: wireless pendants, concealed sensors, DECT handsets, duress code on electronic locks (opens the door but silently alerts security). Best for: staff in public-facing or exposed positions.

Silent alarm = alerts security personnel rather than sounding an audible alarm — allows response and interception before the intruder knows they have been detected. Closed-circuit alarms are more secure than open-circuit — cutting the wire in an open-circuit alarm defeats it. Dual-technology sensors (microwave + infrared) significantly reduce false positives because both must trigger simultaneously.

Piggybacking is slipping through before the door closes. The mantrap prevents both by requiring the first door to fully close before the second opens — making it physically impossible to piggyback or tailgate.

ADeadbolt lock
BElectronic keypad lock

The credential cannot be lost (it is part of the person), stolen (it cannot be copied in normal circumstances), shared, or forgotten. Physical keys can be copied. PINs can be shared or forgotten. What You Have Learned

📦

Asset Management

Identify, classify, track, and protect all IT assets — hardware, software, data, and personnel. The foundation of every other security function.

🔍

Asset Tracking Methods

Manual inventory, network scanning, asset management software, CMDB, MDM, cloud asset discovery. RFID tags enable real-time location tracking.

⚙️

Configuration Management

Configuration Items, baseline configurations, CMS, CMDB, change control. ITIL framework. Unauthorized changes are a major source of misconfigurations.

💾

Data Backups

On-site, off-site, and air-gapped. Enterprise features: deduplication, instant recovery, ransomware protection. Recovery validation testing is mandatory.

📸

Advanced Data Protection

VM, filesystem, and SAN snapshots. Synchronous and asynchronous replication. Journaling for granular recovery. Backup encryption for compliance.

🗑️

Secure Data Destruction

HDD: multi-pass overwriting. SSD: ATA Secure Erase. Sanitization for repurposing. Physical destruction for decommissioning. Third-party certification for compliance.

📋

COOP & Business Continuity

COOP = immediate response to critical function disruption. BC = broader long-term organizational resilience. COOP is a component of BC.

📊

High Availability

The nines — 99% through 99.9999%. MTD, fault tolerance, redundancy. Hot, warm, cold, and cloud recovery sites. Scale-up vs. scale-out.

🔗

Clustering

Virtual IP, Active/Passive, Active/Active, N+1, N+M. Application clustering shares session state. Load balancers distribute traffic — clusters share data.

⚡

Power Redundancy

Dual PSUs, managed PDUs, UPS (always required), backup generators. UPS bridges the gap between power failure and generator activation.

🕵️

Deception Technologies

Honeypots, honeynets, honeyfiles, honeytokens, fake telemetry, DNS sinkholes. Raise the cost of reconnaissance — waste attacker resources.

🏢

Physical Security

Authentication, authorization, accounting. Security zones. Fencing, lighting, bollards. Mantrap, biometric locks, PACS. CCTV, AI surveillance, alarms, and sensors.

Key Terminology

Lesson 7 Glossary

Essential terms and definitions for the CompTIA Security+ SY0-701 exam.

Air-Gapped Backup
A backup physically disconnected from the network — cannot be reached by ransomware under any circumstances. Gold standard for ransomware resilience.
ATA Secure Erase
An SSD-specific command that instructs the drive's firmware to internally sanitize all stored data, including inaccessible marked-as-bad memory cells.
Bollard
Short vertical post — steel or concrete — installed at intervals around a perimeter to protect against vehicular attacks and prevent unauthorized vehicle access.
Business Continuity (BC)
Broader than COOP — addresses overall organizational resilience including supply chain, employee safety, legal compliance, and reputation management.
Cable Lock
A physical lock that attaches to a device's Kensington security slot or metal loop, securing it to a rack or desk and preventing removal or chassis opening.
Certification (Data Destruction)
Documentation and formal verification from a reputable third-party provider attesting that data was securely destroyed in accordance with industry standards.
CMDB
Configuration Management Database — centralized repository tracking assets, configurations, and relationships between systems. Tools: ServiceNow, BMC Remedy.
COOP
Continuity of Operations Planning — processes and procedures ensuring critical business functions continue during and after a disruption. Focuses on immediate response.
Data Deduplication
Identifies and eliminates redundant data blocks — stores one copy and creates references to all other instances. Reduces storage requirements and speeds data transfer.
DNS Sinkhole
Routes suspect traffic to a different network (such as a honeynet) where it can be analyzed, rather than allowing it to reach its intended destination.
Duress Alarm
Manually triggered by staff under threat — allowing silent alerts to security while the staff member appears to cooperate normally with an intruder.
Fault Tolerance
A system that continues providing the same level of service even when components fail. Achieved by provisioning redundancy for critical components and single points of failure.
Honeyfile
Fake files that appear to contain sensitive information — payroll, passwords, customer data. Any access triggers an immediate alert detecting unauthorized activity.
Honeynet
A network of interconnected honeypots simulating an entire organization's network. Provides a richer environment for observing multi-stage attacker behavior.
Honeypot
A decoy system designed to mimic a real server or service. Any interaction is suspicious. Monitors attacker behavior and gathers intelligence on TTPs.
Honeytoken
False credentials, API keys, or login tokens embedded in systems or files. When an attacker steals and uses one, it triggers an immediate alert.
Hot Site
An alternate processing site that can fail over almost immediately — fully operational with current data. Most expensive recovery site type.
Journaling
Records every change to data in a separate, dedicated log. Enables recovery after system crashes by identifying and undoing incomplete transactions. Examples: NTFS, JFS.
Kensington Security Slot
Standardized security slot found on laptops, servers, and monitors for cable lock attachment — prevents physical theft and chassis opening.
Mantrap
Access control vestibule — two interlocked doors permitting only one individual at a time. Prevents tailgating and piggybacking. Used in data centers and financial institutions.
MTD
Maximum Tolerable Downtime — the availability requirement for a specific business function. How long it can be down before the organization suffers unacceptable harm.
N+1 Configuration
A single passive node shared among multiple active nodes. Reduces passive node costs — one passive node serves as backup for any one of several active nodes.
PACS
Physical Access Control System — combines access badges, card readers, control panels, and centralized software to manage badge-based access with comprehensive logging.
PIR Sensor
Passive Infrared sensor — detects moving heat sources such as a human body entering a monitored area. Used in motion detection alarm systems.
RFID Tag
A chip programmed with asset data. When in range of a scanner, the chip activates and signals the scanner — enabling real-time location tracking and theft deterrence.
Sanitization
Removing sensitive information from storage media so it becomes irretrievable — even with specialist recovery tools. Used when repurposing or donating devices.
Security Baseline
The minimum set of security configuration settings a device or software must maintain to be considered adequately protected. Any deviation is a finding requiring remediation.
Security Zone
A defined physical area separated from adjacent areas by barriers with controlled entry and exit points. Progression through zones must become increasingly restrictive.
Tailgating
Following an authorized person through a secured door without independent authentication. Prevented by access control vestibules (mantraps).
TCO
Total Cost of Ownership — the full cost of an asset including initial purchase, maintenance, update costs, and potential incident costs. Used in secure procurement decisions.
UPS
Uninterruptible Power Supply — provides temporary power during a complete power failure. Always required alongside generators — generators cannot respond fast enough on their own.
Virtual IP (VIP)
A single public IP address shared between clustered appliances — also called a floating or shared address. Maintained via a redundancy protocol such as CARP with a heartbeat mechanism.
