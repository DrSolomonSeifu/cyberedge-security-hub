🚨 Why This Lesson Is Your Foundation

On May 7, 2021, Colonial Pipeline — the company supplying nearly half of the East Coast's fuel — was shut down by a ransomware attack. The attacker used a single compromised password to access the network through a VPN account that lacked multi-factor authentication.

No advanced zero-day exploit. No nation-state magic. Just one leaked password, and a missing basic control.

Every concept in this lesson — CIA Triad, access control, security controls, frameworks — exists to prevent exactly this kind of failure. Master this lesson and you will understand why Colonial Pipeline happened — and what should have stopped it.

📖 Study Content — Lesson 1

🔐 The CIA Triad & Non-Repudiation

Information security — shortened to infosec — protects data from unauthorized access, attack, theft, or damage. Data is vulnerable at every stage: while stored, while transmitted, and while being processed.

🔒 Confidentiality

Information can only be read by explicitly authorized individuals. Unauthorized access — regardless of intent — must be prevented.

Example: Patient records visible only to treating physicians.

✅ Integrity

Data is stored and transferred exactly as intended. Any modification must be explicitly authorized through proper channels.

Example: A $500 transaction must arrive as $500 — not altered in transit.

📡 Availability

Information must be readily accessible to those authorized to view or modify it. Locking out legitimate users is a failure of availability.

Example: An ER physician needs patient records immediately — downtime is dangerous.

📝 Exam Note: The CIA Triad is also called AIC (Availability, Integrity, Confidentiality) to avoid confusion with the Central Intelligence Agency. Know both — the exam may use either term.

Non-Repudiation — The Fourth Property

Non-repudiation means a person cannot deny having performed an action — creating, modifying, or sending a resource. It provides proof of origin and proof of action.

Real-World Example: A legally witnessed will cannot be disputed by claiming it was never signed — the witness provides the non-repudiation evidence. In digital environments, digital signatures and audit logs enforce non-repudiation.

🏛️ The NIST Cybersecurity Framework — Five Core Functions

NIST (National Institute of Standards and Technology) developed one of the most widely adopted cybersecurity frameworks. It organizes all cybersecurity activity into five core functions that operate continuously and simultaneously.

🔍 Identify

Develop security policies. Evaluate risks, threats, and vulnerabilities. Recommend controls.

🛡️ Protect

Procure, deploy, operate, and decommission IT assets with security embedded at every stage.

🔎 Detect

Perform continuous, proactive monitoring to identify threats before they escalate.

⚡ Respond

Identify, analyze, contain, and eradicate threats to systems and data.

🔄 Recover

Restore systems and data after an attack. Build resilience to bounce back faster.

⚠️ Important: These five functions are not sequential steps. They operate simultaneously within a mature security program. Detection feeds back into Identification. Recovery strengthens Protection. Think of them as a continuous cycle — not a checklist.

📊 Gap Analysis

A gap analysis compares what an organization currently has in place against what a framework requires or recommends. It answers one fundamental question: where does our current security posture fall short?

When to Perform a Gap Analysis

  • When first adopting a new security framework.
  • When a new regulation or compliance requirement takes effect.
  • Periodically — every few years — to validate changes and confirm compliance.

What a Gap Analysis Report Produces

  • An overall capability score for each section of the framework.
  • A detailed list of missing or poorly configured controls.
  • Specific recommendations for remediation.
Color-Coded Capability Levels: 🟢 Green = Advanced capability  |  🟡 Yellow = Intermediate capability  |  🔴 Red = No or basic capability

Internal vs. External Consultants

Gap analysis can be handled internally — but complex frameworks often require third-party consultants who bring an objective perspective and current knowledge of best practices.

🔑 Access Control and Identity and Access Management (IAM)

Access control ensures a system upholds the CIA Triad by governing how subjects interact with objects.

  • Subjects — entities requesting access: people, devices, software processes.
  • Objects — resources being accessed: networks, servers, databases, files.

The Four IAM Processes (AAA + Identification)

ProcessWhat It DoesExample
IdentificationCreates a unique account or ID to represent a user, device, or process.IT creates a new employee's user account.
AuthenticationProves the subject is who it claims to be using an authentication factor.Entering a username and password to log in.
AuthorizationDetermines what rights the subject has on each resource and enforces them.Employee can read a file but cannot write to it.
AccountingTracks all actions in an audit log; creates a non-repudiation trail.Log records that User 101 accessed a file at 2:14 AM.
AAA vs. IAM: AAA (Authentication, Authorization, Accounting) is an older term. IAM is increasingly preferred because it also acknowledges the critical first step — Identification. Both describe the same system.

🛡️ Security Control Categories

Security controls are safeguards that give systems the properties of the CIA Triad and non-repudiation. Controls are divided into four categories based on how they are implemented.

📋 Managerial

Provide oversight and governance. They govern how security decisions are made — not how attacks are blocked. Examples: risk identification processes, control selection tools.

👤 Operational

Implemented primarily by people. Depend on human action and judgment. Examples: security awareness training, security guards, workstation lock policies.

💻 Technical

Implemented through hardware, software, or firmware. Operate automatically. Examples: firewalls, antivirus software, OS access control models.

🏢 Physical

Govern access to physical premises and hardware. Deter and detect physical intrusion. Examples: cameras, locks, alarms, security lighting, security personnel.

The Control Pyramid: Managerial sits at the top (strategy). Operational sits in the middle (people). Technical and Physical share the base (frontline). All four layers must work together — no single category is sufficient on its own.

⚙️ Security Control Functional Types

Controls can also be classified by what they are designed to accomplish — their functional type. A single control belongs to both a category and a functional type simultaneously.

🔒 PreventiveBefore the attack
🔎 DetectiveDuring the attack
🔧 CorrectiveAfter the attack

The Three Primary Functional Types

  • Preventive — Eliminates or reduces the likelihood of a successful attack. Operates before the attack. Examples: firewall ACLs, antimalware software, proactive patch management.
  • Detective — Identifies and records an intrusion — but does not stop it. Operates during the attack. Examples: logs, intrusion detection systems.
  • Corrective — Reduces the impact of a policy violation or attack. Operates after the attack. Examples: backup and recovery systems.

Three Additional Functional Types

  • Directive — Enforces rules of behavior through policy, procedure, or training. Examples: employee contracts with disciplinary terms, security awareness programs.
  • Deterrent — Discourages attackers psychologically without physically blocking access. Examples: warning signs, visible cameras. The attacker chooses a different target.
  • Compensating — A substitute for a primary control that cannot be implemented. Must provide equivalent or better protection through a different method. Example: network segmentation to isolate a system that cannot be patched.
⚠️ Exam Nuance: Patch management is typically preventive. However, if applied after a vulnerability has already been exploited, it becomes corrective. Context determines the functional type.

👥 Information Security Roles and Responsibilities

A security policy is a formalized written statement defining how an organization protects its data. Every employee — from the executive level to non-technical staff — has a defined role in the security posture.

CIO — Chief Information Officer

Holds overall responsibility for the IT function. May carry direct responsibility for security. Larger organizations may also appoint a CTO focused on emerging technologies.

CSO / CISO — Chief Security Officer / Chief Information Security Officer

Leads the dedicated security function in larger organizations. Responsible for design, implementation, and oversight of the entire security program.

Managers

Hold domain-level security responsibility — building control, web services, accounting, etc. Translate organizational policy into day-to-day practice within their teams.

ISSO — Information Systems Security Officer

Technical specialist responsible for implementing and enforcing security policy. A dedicated security administrator title within the technical/specialist staff category.

Non-Technical Staff

Responsible for complying with policies and applicable laws. Their behavior — handling data, following procedures, reporting anomalies — directly impacts the security posture.

Due Care and Liability (External)

External accountability rests primarily with directors and owners. Failure to exercise due care can result in legal liability. All employees share some measure of responsibility.

🎯 Information Security Competencies

Security professionals must be competent across a wide range of disciplines — from technical network design to non-technical areas like procurement and human resources.

  • Risk assessments and testing — evaluate systems, identify weaknesses, make actionable recommendations.
  • Specifying, sourcing, installing, and configuring secure devices and software — security is embedded from procurement through deployment.
  • Access control and user privilege management — implement least privilege; users get only what they need.
  • Auditing logs and reviewing access — detect anomalies, review privilege assignments, document controls.
  • Incident response and reporting — identify, contain, eradicate, recover, and report on incidents.
  • Business continuity and disaster recovery — create and test plans; an untested plan is an untested assumption.
  • Security training and education programs — human behavior is consistently a major factor in security incidents.

🏢 Information Security Business Units

SOC — Security Operations Center

A dedicated facility where security professionals monitor and protect critical information assets 24/7. Enables rapid incident detection and response across all business functions. Resource-intensive — typically found in larger organizations such as government agencies, healthcare companies, and financial institutions.

DevSecOps

DevOps integrates development and IT operations to break down silos. DevSecOps extends this by embedding security specialists into every stage of development and operations from the very beginning. This principle is also known as shift left — security must be addressed during requirements and planning, not grafted on at the end.

CIRT / CSIRT / CERT

A dedicated team serving as the single point of contact for all security incidents. Receives notifications, coordinates the response, and manages recovery. May be part of the SOC or operate as an independent business unit.

Know All Three Names: Computer Incident Response Team (CIRT) | Computer Security Incident Response Team (CSIRT) | Computer Emergency Response Team (CERT). All three describe the same core function.

📖 Glossary of Key Terms — Lesson 1

A
Audit Log

Accounting / Monitoring

A recorded history of system activity used to detect unauthorized access or policy violations. Audit logs create a non-repudiation trail — users cannot later deny actions that have been logged.

AAA (Authentication, Authorization, and Accounting)

Identity & Access Management

The three core operational functions within an IAM system. Sometimes used interchangeably with IAM, though IAM is increasingly preferred because it also acknowledges the Identification step.

Availability

CIA Triad

The property ensuring that information is readily accessible to those authorized to view or modify it. A system so locked down that legitimate users cannot access their data has failed on availability.

B
Business Continuity

Resilience

The capability of an organization to maintain essential functions during and after a disruption. Distinguished from disaster recovery — continuity focuses on keeping operations running; recovery focuses on restoring them.

C
CIA Triad (also AIC)

Core Security Concept

The three foundational properties of secure information: Confidentiality, Integrity, and Availability. Sometimes called AIC to avoid confusion with the Central Intelligence Agency. Both terms refer to the same three properties.

CIO (Chief Information Officer)

Security Roles

Holds overall responsibility for the IT function across the organization. In some organizations, the CIO also carries direct responsibility for security.

CIRT / CSIRT / CERT

Security Business Units

Computer Incident Response Team / Computer Security Incident Response Team / Computer Emergency Response Team. Three names for the same core function: serving as the single point of contact for managing security incidents.

CSO / CISO

Security Roles

Chief Security Officer / Chief Information Security Officer. Leads the dedicated security function in larger organizations, responsible for the design, implementation, and oversight of the entire security program.

Compensating Control

Control Functional Types

An alternative control used when the primary recommended control cannot be implemented. Must provide the same level of protection — or better — through a different method or technology. Example: network segmentation to isolate an unpatched legacy system.

Compliance

Governance

Meeting the security requirements imposed by applicable laws, regulations, or contractual obligations. Non-compliance is both a security risk and a legal liability.

Confidentiality

CIA Triad

The property ensuring that information can only be read by people who have been explicitly authorized to access it. Unauthorized individuals — regardless of intent — should never be able to view protected data.

Continuous Monitoring

NIST Framework

The ongoing, proactive process of observing systems and controls to ensure they remain effective against current and emerging threats. Supports the Detect function of the NIST framework.

Corrective Control

Control Functional Types

A control that reduces the impact of a security policy violation or attack. Operates after the attack. Example: a backup and recovery system that restores data damaged during an intrusion.

Cybersecurity Resilience

NIST Framework

The capacity to recover and continue operating after a successful attack. Supports the Recover function of the NIST framework.

D
Detective Control

Control Functional Types

A control that identifies and records an intrusion attempt or success. Operates during the attack — it does not stop access but captures what is happening. Example: system logs and intrusion detection systems.

Deterrent Control

Control Functional Types

A control that psychologically discourages attackers from attempting an intrusion without physically blocking access. Example: visible security cameras and legal warning signs — the attacker chooses a different target.

DevSecOps / DevOps / Shift Left

Security Business Units

DevOps integrates software development and IT operations. DevSecOps extends this by embedding security at every stage of development and operations. Shift left means security must be addressed during requirements and planning — not added at the end.

Directive Control

Control Functional Types

A control that enforces rules of behavior through policies, best practice standards, or standard operating procedures. Sets expectations and defines consequences for non-compliance. Example: employee contracts with disciplinary terms.

Disaster Recovery

Resilience

The processes and procedures for restoring systems and data following a major incident. A recovery plan that has never been tested is an untested assumption — not a real safeguard.

Due Care

Security Roles

The legal and ethical obligation of directors and owners to take reasonable and appropriate steps to protect the organization and those it serves. Failure to exercise due care can result in legal liability.

F
Framework (IT Service Framework)

Governance

A structured set of guidelines that organizations use to build and evaluate their security programs. Saves organizations from building security programs in isolation by providing a proven, structured foundation. Examples: NIST CSF, ISO 27001, CIS Controls.

G
Gap Analysis

Governance

The process of identifying how an organization's current security capabilities deviate from framework requirements or recommendations. Produces a capability score, a list of missing controls, and remediation recommendations.

H
Hardening

Security Controls

The process of reducing a system's attack surface by eliminating unnecessary services, applying patches, and enforcing configuration standards. Part of the Protect function in the NIST framework.

I
IAM (Identity and Access Management)

Identity & Access Management

The system that manages the full lifecycle of digital identities and their access to resources. Encompasses Identification, Authentication, Authorization, and Accounting. The modern preferred term over AAA because it acknowledges the Identification step.

Incident Response

Security Competencies

The structured process of identifying, containing, eradicating, and recovering from a security incident. Accurate and timely reporting is critical — both for internal decisions and for legal or regulatory obligations.

Infosec (Information Security)

Core Security Concept

The protection of data resources from unauthorized access, attack, theft, or damage. Data is vulnerable while stored, while in transit, and while being processed — making infosec a multidimensional discipline.

Integrity

CIA Triad

The property ensuring that data is stored and transferred exactly as intended. Any modification must be explicitly authorized through proper channels. Unauthorized changes — even small ones — compromise the trustworthiness of the entire dataset.

ISSO (Information Systems Security Officer)

Security Roles

A technical specialist responsible for implementing and enforcing security policy. A dedicated security administrator title within the technical/specialist staff category.

IT Service Framework

Governance

A structured best practice guide used to govern IT and cybersecurity functions in alignment with business needs. Provides checklists of procedures, activities, and technologies that represent best practice.

L
Least Privilege

Access Control

The principle that users and systems should have access only to what is necessary to perform their assigned function — nothing more. Directly supports the Authorization component of IAM.

Liability

Security Roles

The external accountability that falls on organizational leadership when due care obligations are not met. Directors and owners bear primary external responsibility for the organization's security posture.

M
Managerial Control

Control Categories

A control that provides governance and oversight of the security program. Governs how security decisions are made and evaluated — does not directly block attacks. Sits at the top of the control pyramid.

N
NIST Cybersecurity Framework

Frameworks

A widely adopted structure developed by the National Institute of Standards and Technology that organizes all cybersecurity activity into five functions: Identify, Protect, Detect, Respond, and Recover. These functions operate continuously and simultaneously.

Non-Repudiation

Core Security Concept

The assurance that an action or event cannot be denied after the fact. Provides proof of origin and proof of action. Enforced through digital signatures and audit logs in digital environments.

O
Operational Control

Control Categories

A control that depends on human implementation and behavior to be effective. Examples include security awareness training programs and security guards. Sits in the middle of the control pyramid.

Object

Access Control

The resource being accessed in an access control system. Objects can include networks, servers, databases, applications, and files. Subjects are assigned rights or permissions on objects.

P
Physical Control

Control Categories

A control that governs access to physical locations and tangible assets. Deters and detects physical access rather than digital access. Examples: security cameras, locks, alarms, lighting, security personnel. Shares the base of the control pyramid with technical controls.

Preventive Control

Control Functional Types

A control that eliminates or reduces the likelihood of a successful attack. Operates before the attack occurs. Examples: firewall ACLs, antimalware software, proactive patch management, SOPs.

R
Regulatory Compliance

Governance

The demonstration that an organization meets the security requirements imposed by applicable laws or industry regulations. Non-compliance is both a security risk and a legal liability.

Remediation

Governance

The corrective action taken to close a gap between an organization's current security capability and the required standard. Gap analysis reports include specific remediation recommendations for each identified gap.

S
Security Control

Core Security Concept

A safeguard or countermeasure designed to give a system or data asset the properties of confidentiality, integrity, availability, and non-repudiation. Controls are classified by category (how implemented) and functional type (what they accomplish).

Security Policy

Governance

A formalized, written statement that defines how an organization will implement and maintain security. It is not a suggestion — it is the official standard against which security decisions and behaviors are measured.

Security Posture

Governance

The overall strength and effectiveness of an organization's security program. An organization with clear security policies and framework-based controls has a strong security posture.

SOC (Security Operations Center)

Security Business Units

A centralized facility — physical or virtual — where security professionals monitor and protect critical information assets continuously. Enables rapid incident detection and response across all business functions. Typically found in larger organizations due to high resource requirements.

Subject

Access Control

Any entity that requests access to a resource in an access control system. Subjects can be people, devices, software processes, or any other system that can be granted access to a resource.

T
Technical Control

Control Categories

A control implemented through hardware, software, or firmware. Operates automatically or semi-automatically without requiring human intervention for each action. Examples: firewalls, antivirus software, OS access control models. Shares the base of the control pyramid with physical controls.
